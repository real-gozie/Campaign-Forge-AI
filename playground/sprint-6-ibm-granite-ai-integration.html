IBM Granite AI Integration

Live AI reasoning layer wired into Campaign Forge AI via Cloudflare Worker proxy. File: campaign-forge-ai.html · Full fallback guarantee. IBM API key never exposed to the browser.

How to Connect IBM Granite

1. Deploy the WorkerFollow cloudflare-worker/DEPLOY.md — run wrangler deploy, set IBM_API_KEY secret and WATSONX_PROJECT_ID in wrangler.toml
2. Open wizardClick "Start Forging" on the homepage
3. Click ⚙ AI SettingsButton in the wizard nav bar (top-right)
4. Cloudflare Worker URLPaste your *.workers.dev URL — e.g. https://campaign-forge-ai-proxy.YOUR-SUBDOMAIN.workers.dev
5. Modelibm/granite-3-8b-instruct (recommended) — select from dropdown
6. Test ConnectionClick "Test Connection" — pings the Worker's /test endpoint which verifies IAM + Granite end-to-end
7. SaveConfig persists in sessionStorage — button turns green "✓ Granite Connected"

Data Flow Architecture

1
User completes Guided Campaign Wizard (5 steps · 32 fields)
All inputs stored in formData{} and tagSelections{} — campaign name, sector, goal, problem, audience, channels, tone, KPIs, activities, partners, risks, and 20+ more fields.
2
buildGranitePrompt() — 28-field structured prompt construction
All formData and tagSelections fields serialised into a clean labelled list. A system prompt instructs Granite to behave as a 5-expert consulting team and return a single structured JSON object with 12 top-level keys.
3
callWorkerProxy() — single POST to the Cloudflare Worker
POSTs { input, model, parameters } to GRANITE_CONFIG.workerUrl + "/analyze". The Worker handles IBM IAM token exchange and watsonx.ai forwarding server-side. The browser never sees the IBM API key. 90s timeout.
4
Cloudflare Worker — IAM token exchange + Granite inference
Worker exchanges IBM_API_KEY (Cloudflare Secret) for a bearer token via IBM IAM, then POSTs to WATSONX_ENDPOINT/ml/v1/text/generation. Returns { result: "..." }. Token cached in Worker isolate for 55 minutes.
5
parseGraniteResponse() — robust JSON extraction
Strips markdown fences, finds outer { } boundaries, parses JSON, and repairs common issues (trailing commas). Returns null on failure — triggering fallback.
6
AI data populates all output surfaces
Expert Review cards, Forge Insights, Expert Collaboration thread, Team Recommendation, Campaign Builder deliverables — all replaced with AI content when aiGeneratedData is set.

Zero-disruption fallback guarantee: If Granite credentials are not configured, if the IAM token exchange fails, if the Granite API returns an error, or if the JSON cannot be parsed — the product silently falls back to the high-quality static expert templates built in Sprints 3–5. The user experience is identical in either mode. The AI loading strip shows the error message only in the analysis-transition screen, and only briefly. No screen is broken, no navigation is disrupted.
